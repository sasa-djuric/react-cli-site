# Project config

<Description>Project configuration object</Description>

Project configuration object describes project preferences. It will be merged internally in each configuration entry.

| Option                  | Type                        | Description                                                              |
| ----------------------- | --------------------------- | ------------------------------------------------------------------------ |
| <code>path</code>       | <code>string</code>         | Project source path                                                      |
| <code>typescript</code> | <code>boolean</code>        | Uses typescript                                                          |
| <code>lint</code>       | <code>boolean</code>        | Uses eslint to lint created files if eslint is used in the project       |
| <code>format</code>     | <code>boolean</code>        | Uses prettier to format created files if prettier is used in the project |
| <code>verbose</code>    | <code>boolean</code>        | Outputs operation and file path to stdout                                |
| <code>fileNaming</code> | [File naming](#file-naming) | Specify file naming preferences                                          |
| <code>export</code>     | [Export](#export)           | Specify export preferences                                               |

## File naming

| Option              | Type                                                                                                 | Description                                                                           |
| ------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| <code>name</code>   | <code>string</code>                                                                                  | Specify name. [Name placeholders](#name-placeholders) are available for usage in name |
| <code>casing</code> | <Space><code>camel</code> \| <code>pascal</code> \| <code>kebab</code> \| <code>snake</code></Space> | Specify casing type                                                                   |

### Name placeholders

| Type                | Description                 |
| ------------------- | --------------------------- |
| <code>{name}</code> | Provided name               |
| <code>{type}</code> | File type (style, story...) |

## Export

| Option               | Type                 | Description                              |
| -------------------- | -------------------- | ---------------------------------------- |
| <code>default</code> | <code>boolean</code> | Specify export type (default or named)   |
| <code>inline</code>  | <code>boolean</code> | Specify should export be declared inline |
